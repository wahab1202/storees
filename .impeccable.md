# Storees — Design Context

## Users
Enterprise marketing teams at companies using ecommerce (Shopify, Medusa), fintech, and SaaS platforms. They manage customer segments, build automated flows, run multi-channel campaigns (email, SMS, push, WhatsApp), and analyze engagement data. They need sophisticated, reliable tools that inspire confidence — not toys. They work in Storees daily and need information density without cognitive overload.

## Brand Personality
**Professional, intelligent, reliable** — like MoEngage or HubSpot. Clean, serious, trustworthy. The interface should feel like a precision instrument, not a playful app. Every element should look intentional and earn its place.

## Aesthetic Direction
- **Tone**: Refined minimalism with strategic use of indigo accent (#4F46E5) against soft blue-gray surfaces (#F5F6FF)
- **References**: MoEngage dashboard, HubSpot campaign builder, Linear's precision, Amplitude's data density
- **Anti-references**: Overly playful UI (Mailchimp illustrations), dark-mode-everything, gradient text, glassmorphism, bouncy animations
- **Theme**: Light mode. Dark navy sidebar (#0D1138) with white content area. Clean borders, subtle shadows.
- **Typography**: Inter — clean, professional, highly readable at small sizes. No decorative fonts.

## Design Principles

1. **Information density over decoration** — Enterprise users want to see data, not whitespace. Pack information meaningfully without clutter. Use progressive disclosure for complexity.

2. **Consistency is trust** — Same patterns everywhere. Same spacing, same border radius, same button styles. Inconsistency erodes confidence in a tool that handles customer data and campaigns.

3. **Quiet confidence** — No flashy gradients, no bouncing animations. Smooth, fast transitions (150-250ms ease-out). The UI should feel calm and under control, even when showing complex data.

4. **Every pixel earns its place** — No decorative elements that don't serve function. Icons convey meaning, not decoration. Color signals status, not aesthetics. Whitespace creates hierarchy, not filler.

5. **Show, don't tell** — Prefer data visualization over text descriptions. Inline metrics over modal details. Live previews over static labels. The interface should teach itself through progressive interaction.

## Existing Design Tokens
- Primary accent: `#4F46E5` (indigo)
- Sidebar: `#0D1138` (dark navy)
- Page bg: `#F5F6FF` (light blue-gray)
- Text: `#1A1A2E` (primary), `#6B7280` (secondary), `#9CA3AF` (muted)
- Border: `#E5E7EB`
- Font: Inter, system-ui fallback
- Spacing: 4px grid (Tailwind)
- Border radius: `rounded-lg` (8px) for cards, `rounded-md` (6px) for inputs
- Shadows: `shadow-sm` for cards, no heavy shadows
